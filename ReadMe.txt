  このパッケージの説明:

    このパッケージには、以下のファイルが含まれています。

      readme.txt      いま、あなたが読んでいるファイル

      ca-bundle.crt   セキュリティ証明書 (https で通信するときに使用します)

      blogolee.exe    ブログクライアント

      xmlRPC.dll      上記のプログラムを動かすのに必要なモジュール (サーバとの通信用)
      iconv.dll       同上 (文字コードの変換用) [このファイルの配布は GNU LGPL に従います]

      blogolee.exe.manifest
                      blogolee.exe を Windows XP 以降で動かす場合に参照するファイル
                      (ただし、Windows をクラシック設定にしている場合は、Windows XP 以降の
                       場合でも参照されません)

      iconv-1.9.1.win32 以下のファイル
        Windows 用 の GNU iconv ライブラリ のバイナリパッケージ
        (GNU LIBRARY GENERAL PUBLIC LICENSE に基づく配布です)

      src 以下のファイル
        blogolee.exe のソースファイル
        (xmlRPC.dll のソースファイルは今回の配布対象には入れていません)

    それ以外のファイル(blogolee.exe, xmlRPC.dll, および src 以下のファイル)は、tsupo の著作物です。配布は自
    由としますが、プログラムの動作および動作に伴い発生する諸々の事象に関しては無保証とします。

    書籍、雑誌等で「ブログクライアント BloGolEe」の紹介を行なったり、本パッケージを収録したりする場合は、事
    前に連絡してください。

    また、src/include 以下のヘッダファイルの内容を元に xmlRPC.dll の動作を推定し、xmlRPC.dll を利用するプロ

    態をできるだけ避けたいのが理由の1つです) xmlRPC.dll を利用するプログラムを作成した、もしくは作成しようと
    していることを連絡していただければ、当方の作業に支障のない範囲で xmlRPC.dll に関する最新の情報を提供する
    ことも可能です。

  インストール:
    本アーカイブに収録されている blogolee.exe, xmlRPC.dll, iconv.dll, ca-bundle.crt を同一のディレクトリ
    (フォルダ)に解凍し、ご利用ください。特に、必要な作業はありません。

  アンインストール:
    blogolee.exe, xmlRPC.dll, iconv.dll, ca-bundle.crt, および setting ディレクトリ(フォルダ) を丸ごと削
    除するだけです。レジストリはいじっていません。なお、setting ディレクトリには、blogolee.exe が生成した
    各種設定ファイルや作業用のファイルが格納されています。

  使い方:
    blogolee.exe
      アイコンをマウスでダブルクリックすると起動します。
      ・設定ファイルが何もない状態で起動すると、blog の登録画面になります。「blog 追加」ボタンを押すと、
       「blog 追加・登録画面」が表示されますので、まず、左の欄に blog にログインするときのユーザ名(また
        はメールアドレス)、パスワード、blog の種類を入力してください。「詳細情報取得」ボタンを押すと、
        blog の名前や エンドポイント URL などを自動取得します(「blog の種類」によっては、「詳細情報取得」
        ボタンを押す必要のない場合もあります)。なお、Movable Tyoe(3.x以降)、So-net blog、Windows Live 
        Spaces など、blog にログインするときのパスワードではなく、API 投稿用、モバイル投稿用、メール投稿
        用など、別途用意されているパスワード(もしくは、それに類するもの)を使う必要がある blog もあります。
        ログイン用のパスワードを使ってもうまく情報が取得できない、あるいは記事が投稿できない場合は、ログ
        イン用のパスワードとは別に用意されているパスワードを使う必要がある可能性があります。
        
        また、ドリコムブログのように、XML-RPC API 利用時のアカウント名(ユーザ名)自体が別途用意されている
        場合もあります。ご注意ください。

        複数の blog を登録したい場合は、「blog の追加」ボタンを押せば、どんどん追加していけます。
        「blog の削除」や「登録情報修正」もできますので、削除したいときや、修正したいときは、それぞれ
        ボタンを押して、作業を進めてください。

        blog の設定が終わると、「ブログクライアント BloGolEe」のメイン画面になります。

      ・すでに設定ファイルがある場合は、「ブログクライアント BloGolEe」のメイン画面がすぐに表示されます。 

      以下、「ブログクライアント BloGolEe」のメイン画面の使い方です。

      基本的に、記事の題名、本文(それ以外の項目は書かなくても構いません)を書いて、「投稿」ボタンを押す
      と、記事を投稿します。 blog によっては、カテゴリの取得、設定も可能です。

      「はてなダイアリーキーワード」や「おすすめ」を利用した記事の投稿も可能です。

      「はてなダイアリーキーワード」を使うと、記事の中に、はてなダイアリー(http://d.hatena.ne.jp/)で用意
      されているキーワードと一致する「ことば」が含まれている場合、その「ことば」に対応する「はてなダイア
      リーキーワード」へのリンクが埋め込まれます。

      「おすすめ」を使うと、投稿しようとしている記事に関連する可能性があるアマゾンの商品と楽天市場の商品
      の紹介を、記事の後ろに掲載することができます。アマゾンのアソシエイトID や楽天Webサービスのアフィリ
      エイトIDを指定すれば、アフィリエイトとしての利用も可能です。ただし、アフィリエイト記事の投稿を禁止
      している  blog もありますので、「おすすめ」を使う場合は、アフィリエイト記事を投稿しても問題ないか、
      あらかじめ確認しておいてください。アソシエイトID やアフィリエイトID が未設定の場合でも「おすすめ」
      は使えますが、その場合は、私(tsupo)のアソシエイトID やアフィリエイトIDが挿入されますので、それが嫌
      な人は、「おすすめ」は使わない方がいいでしょう。

      「blog 切替」ボタンを押すと、登録済みの blog 一覧が表示されます。この画面で新しい blog の追加登録をす
      ることもできます。投稿対象の blog を選択した後、OK ボタンを押すと、選択した blog に切り替わります。

      まっさらな状態に戻して、最初から設定し直したい場合は、setting ディレクトリの下にあるファイルを全部削除
      して、「ブログクライアント BloGolEe」を起動してください。

     *「おすすめ」機能について (詳細)
         「おすすめ」機能は、投稿しようとしている記事(エントリ)に含まれるキーワード(特徴語)を抽出し、
             (1) そのキーワードを含む書籍、おもちゃをアマゾン(amazon.co.jp)取り扱い商品から検索する機能
             (2) そのキーワードを商品を楽天市場(rakuten.co.jp)取り扱い商品から検索する機能
          の2つの機能から成り立っています。ご自分のアマゾンアソシエイトIDや楽天WebサービスアフィリエイトID
          を取得し、「ブログクライアント BloGolEe」に登録すれば、自動的にアフィリエイト記事を投稿することが
          できます。なお、アマゾンアソシエイトIDや楽天WebサービスアフィリエイトIDの欄が空白のままだと、私
          (tsupo)のアソシエイトID、アフィリエイトIDが使われます。そんなの嫌だ、って人は、「おすすめ」機能を
          使わないようにするか、自分のアソシエイトID、アフィリエイトIDを取得し、登録してください(アソシエイ
          トID、アフィリエイトIDの取得にお金はかかりません)。

         「おすすめ」を使う場合は、アフィリエイトが使える環境であること(プロバイダやブログの規約上問題がな
         いかどうか)を確認してから使ってください。2007年3月7日時点で、当方で把握している範囲で、アフィリエ
         イトが禁止されているブログサービスについては、「おすすめ」機能が使えないようにしてあります。
         Movable Type 等、自分で設置するタイプのブログツールの場合は、設置先サーバの提供元(プロバイダ)の
         規約を確認してください。

         また、ブログツール sb (Serene Bach) はツールの使用規約にアフィリエイトに関する条項が含まれています
         ので、抵触しないかどうか確認してください(sb の場合は、「おすすめ」は使わない方がいいと思われます)。

    *ホットペッパー API について
         メイン画面から、ホットペッパーAPI を利用した「飲食店検索」が使えます。ブログ記事のネタ探しにご利用
         ください。将来、検索結果を元に記事を作成する機能をサポートする予定です。

    *じゃらんWebサービスについて
         メイン画面から、じゃらんWebサービスを利用した「宿泊施設検索」と「温泉検索」が使えます。ブログ記事
         のネタ探しにご利用ください。将来、検索結果を元に記事を作成する機能をサポートする予定です。
         また、バリューコマースに加入していて、じゃらん.net とアフィリエイト契約済みの人は、じゃらん.net の
         MyLink(テキスト自由リンク[宿泊施設向け])用の広告スペースを設置し、その広告スペースの sid と pid を
         本ブログクライアントのメイン画面のSID欄とPID欄に登録すれば、アフィリエイトリンク生成機能が利用でき
         ます。検索条件を設定後、「検索」ボタンの代わりに「アフィリエイトリンク生成」ボタンを押せば、アフィ
         リエイト用のリンク情報を含む画面が表示されます。このリンク情報を記事に貼り付けて投稿してください。
         今のところ、「アフィリエイトリンク生成」に対応しているのは、「宿泊施設検索」のみで「温泉検索」の方
         は対応していません。


  既知の問題:
    ・「はてなダイアリーキーワードリンク」を使うと記事全体が化けることがある。
        → はてなのサーバから受信した XML 自体がすでに化けているようです。
           現在、文字化けを検出できた場合は、強制的にキーワードリンクの使用を中止することで、対処
           しています。文字化けの検出に失敗した場合はそのまま投稿されてしまいます。
    ・blogger、So-net Blog、アメーバブログ、MSN Spaces、(XML-RPC API 経由の場合の)ドリコムブログで
      はカテゴリの取得、設定が API 経由ではできません(MSN Spaces はカテゴリ情報(カテゴリ別のアーカ
      イブファイルの URL や RSS の URL)の取得はやろうと思えばできないこともありませんが、カテゴリの
      設定はできません)。カテゴリの設定を行ないたい場合は、面倒ですが、本ブログクライアントによる記
      事の投稿後、各ブログの管理画面にて、カテゴリの設定を行なってください。なお、So-net Blog に
      関しては、API 経由で投稿しようとしている記事のデフォルトのカテゴリをあらかじめ管理画面にて設
      定しておくこともできます。ドリコムブログに関しては、XML-RPC API を使わない場合は、カテゴリの
      取得、設定が可能です。
    ・Vox に投稿する場合は、タグがカテゴリに相当します。モバイル投稿時に自動的に設定されるタグが(設
      定済みの場合は)自動的に付加されます。

    また、本ブログクライアントは拙作のライブラリ xmlRPC.dll を使用しています。この xmlRPC.dll は、
    元々 XML-RPC API や Atom 出版プロトコルを利用するアプリケーション向けのライブラリとして用意し
    たものですが、現在は各種 Web サービスを利用するためのユーティリティ関数群をはじめとして、拙作
    のアプリケーションで共通利用している雑多な関数を含んでいます。

    xmlRPC.dll内の SSL による通信を行なう関数群で OpenSSL (のスタティックライブラリ)を利用しています。
    OpenSSL については、以下の Webページを参照してください。
         http://www.openssl.org/

    xmlRPC.dll内の文字コード変換関数群は、GNU の libiconv を Windows 向けにビルドしたもの(iconv.dll)
    を利用しています。iconv.dll に関しては、ビルドするのに必要なソースファイル等を、本パッケージに同
    梱しています。
